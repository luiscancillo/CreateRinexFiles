generateRinexFilesJNI is the interface routine with the Java application toRINEX. It is called to generate RINEX files using data acquired from the GNSS receiver (which are stored in raw data files) and parameters passed in arguments.

Parameters
envwith the app environment (Java specific)
methe interface object
infilesPaththe full path to the directory where raw data files are placed
infilesNamean array of strings each one containing the name of a raw data file
outfilesPaththe full path to the directory where RINEX files will be generated
rinexParamsan array of strings each one containing a pair of messaje_type;value
Returns
a text string describing the processing result for generating RINEX files

1- Defines and sets the error logger object

2 -extract rinex header data from params

3 -classify observation and navigation input file

4 -create navigation file(s)

4.1 -extract from raw data files header and navigation data

5 -create observation files

5.1 -create one RINEX file for each ORD file

5.2 -create a unique RINEX file containing observation data from all raw data files

printNavFiles conducts the printing of the RINEX navigation file or files for the navigation data already collected from the raw data files. Note that depending on the RINEX version to be printed it should be printed a unique file, if version 3 requested, or one file for each constellation, if version 2 requested.

Parameters
prinexpointer to the rinex object where header records will be stored
ploga pointer to the logger
outfilesFullPathis the path name where the output file will be created
markNamethe position mark name to be used to name the output file
Returns
0 if no error occurred, or a code (in the set RET_ERR_xxxx values) identifying the error
